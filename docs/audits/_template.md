## How to use this template

1. **Copy, do not edit in place.**
   `cp docs/audits/_template.md docs/audits/<theme>-<scope>-<YYYY-MM>-<auditor>.md`
   Examples of `<theme>`: `docs`, `security`, `perf`, `a11y`, `i18n`,
   `licensing`, `cost`, `release`. `<scope>` is a short slug pinning the
   release, surface, or subsystem under review.

2. **Fill the frontmatter first.** Title, auditor, date, scope, and
   severity scale set the contract for the rest of the report. If you
   cannot name the scope cleanly, the audit is not yet scoped.

3. **Write findings in the canonical findings-backlog format.** Each
   finding row in the table maps one-to-one with an entry in the season
   writers' room backlog (`docs/exec-reports/sNN-writers-room.md`). See
   [`.github/skills/findings-backlog.md`](../../.github/skills/findings-backlog.md)
   for the entry format and the five-state lifecycle (surfaced, logged,
   triaged, dispositioned, closed).

4. **Pin evidence.** File path with line numbers, commit SHA, workflow
   run ID, or copy-pasteable command. "There is a Unicode bug somewhere"
   is a vibe; `Program.cs:1445` is a finding.

5. **Link the audit from the relevant exec-report.** The episode that
   commissioned the audit (or the next one to read it) lists this file
   under "Cross-references" or in the lessons section. An audit nobody
   links to is an audit nobody reads.

6. **File status updates back into this audit when findings ship.**
   Update the `Status` column in the findings table as work progresses
   (`open` -> `triaged` -> `in-progress` -> `shipped` or `wontfix`).
   When all findings reach a terminal state, set the sign-off status to
   `final` and consider whether a follow-up audit is warranted.

7. **ASCII-only.** Before saving, run:

   ```bash
   grep -P "[\x{2018}\x{2019}\x{201C}\x{201D}\x{2013}\x{2014}\x{2026}]" docs/audits/<your-file>.md
   ```

   The command should return nothing. If it returns a line, replace the
   smart punctuation with ASCII equivalents (`'`, `"`, `-`, `--`, `...`)
   per the [`ascii-validation`](../../.github/skills/ascii-validation.md)
   skill.

8. **Sign and stop.** An auditor proposes; an owner disposes. Do not
   land code changes from inside an audit -- hand the remediation
   backlog to Mr. Pitt or the showrunner for scheduling.
