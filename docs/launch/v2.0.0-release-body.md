Picture it: a Tuesday, 8:47 AM. You hit the Espanso trigger that wraps
`az-ai --raw` around a clipboard of gnarled markdown, and clean prose
lands in the text field. Same as last Tuesday. Same as the Tuesday
before. **That is the point.**

v2.0.0 is not the release where everything looks different. It is the
release where everything looks the same, and the floor underneath is
new. Roughly **2,200 lines** of hand-rolled chat, tool, and Ralph
orchestration are gone, replaced by [Microsoft Agent
Framework](https://github.com/microsoft/agent-framework) primitives —
`ChatClientAgent`, `AgentThread`, MAF function-tools. Streaming, tool
calls, Ralph retries, and the squad dispatch all ride on top. Every v1
flag still works. Every env var reads the same. Every exit code is
unchanged. `--raw` output is byte-identical. `.squad/` memory comes
along untouched.

Three things land alongside the migration:

- **Persona routing, wired end-to-end.** `--persona <name>` and
  `--persona auto` now overlay system prompt, tool allow-list, and
  `.squad/history/<name>.md` memory — and persist memory on session
  exit. In earlier v2 previews the flag was parsed and silently
  ignored. Not anymore.
- **Opt-in OpenTelemetry.** `--telemetry` (or `AZ_TELEMETRY=1`) emits
  spans and per-call cost events to stderr. Off by default. Silent
  under `--raw`. Zero allocation on the hot path when disabled.
- **AOT-first shipping.** The linux-x64 native binary is the
  reference artifact. 12.91 MB, 1.456× v1.9.1 — inside Kenny Bania's
  1.5× gate, no waiver. p95 startup regression inside the ≤25%
  budget. `parse-heavy` is *faster* than v1.

## What's new

- **Microsoft Agent Framework runtime** — [ADR-004][adr4], commit
  `0b2e655`. v1 bespoke loop is gone.
- **Persona routing end-to-end** — commits `0b2e655`, `cbcc49b`.
  System prompt + tool allow-list + per-persona memory, persisted.
- **Cost estimator** — `--estimate` (alias `--dry-run-cost`) and
  `--estimate-with-output <n>` predict USD for a prompt without
  making an API call. Short-circuits before credential or endpoint
  resolution. Safe to call from CI budget gates, offline.
  ([FR-015][fr15])
- **Opt-in OpenTelemetry** — `--telemetry` / `--otel` / `--metrics`.
  Suppressed entirely by `--raw`.
- **Connection prewarming** — `--prewarm` opens the Azure OpenAI
  channel in parallel with prompt assembly, buying back the MAF
  cold-start cost on the first interactive token. ([FR-007][fr7])
- **Nine new flags** — `--estimate`, `--estimate-with-output`,
  `--telemetry`, `--otel`, `--metrics`, `--prewarm`, `--config <path>`,
  `--schema <json>` (captured, not yet enforced — see Known
  limitations), `--version --short`.
- **`.azureopenai-cli.json` config reference** — new
  [`docs/configuration-reference.md`](../configuration-reference.md).
- **Perf baseline harness** — `scripts/bench.sh` +
  [`docs/perf-baseline-v2.md`](../perf-baseline-v2.md). First real
  v1↔v2 comparison (50 runs, 2 warmup, linux-x64 AOT +
  framework-dependent).
- **OSS license artifacts** — `NOTICE` and `THIRD_PARTY_NOTICES.md`

## Upgrade from v1

There is no migration step. During the dual-tree window the v1 binary
installs as `az-ai` and v2 as `az-ai-v2`; they read the same env vars,
the same config files, and the same `.squad/` directory. No flag, env
var, or exit code changes behavior for existing `az-ai` callers —
Espanso, AutoHotkey, and the CI scripts keep working. Flip your
triggers from `az-ai` to `az-ai-v2` when you're ready; flip them back
if you need to. See [`docs/migration-v1-to-v2.md`](../migration-v1-to-v2.md)
for the full story and post-cutover v1-pin instructions.

## Known limitations

Honesty about what doesn't work yet. Full list in
[§Known limitations at 2.0.0](../release-notes-v2.0.0.md#known-limitations-at-200).

- **`--schema <json>` wire enforcement is deferred to 2.1.x.** Flag is
  parsed and captured, not yet sent as a `response_format` strict
  schema. Use `--json` + post-validation in the meantime.
- **arm64 release matrix rolls forward to 2.0.1.** If arm64 is your
  primary platform, pin `v1.9.1` until 2.0.1 ships.
- **Windows CI is not yet enabled.** Windows binaries are produced and
  manually validated pre-release; Linux (glibc, musl) and macOS run
  per release.
- **Multi-provider (FR-014) is a spike, not a feature.** v2.0.0
  remains Azure-OpenAI-only.
- **Malformed `.squad.json` persona names surface as an uncaught
  `ArgumentException`** (exit 134) rather than a clean `[ERROR]` line.
  Security is intact — `SanitizePersonaName` rejects path traversal —
  the UX wrap lands in 2.0.1 (FR-021). Keep persona names to
  `[a-z0-9_-]{1,64}`.
- **Homebrew / Scoop / Nix NOTICE bundling** lands in 2.0.1. Tarball
  and container image carry `NOTICE` + `THIRD_PARTY_NOTICES.md`
  in-band at 2.0.0.
