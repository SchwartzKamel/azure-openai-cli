## One-time recovery recipe (you execute, I lack GH auth)

**Update 2026-04-22 07:09 UTC:** The `workflow_dispatch` lever in
`d2dc627` only applies to workflows on `main` — the `release.yml` file
frozen at the v2.0.2 tag (`fd4ddc7`) does **not** have the trigger, so
`gh workflow run release.yml --ref v2.0.2` returns HTTP 422
(`Workflow does not have 'workflow_dispatch' trigger`).

Use `gh run rerun --failed` instead, which reschedules the cancelled /
failed legs against the tag's workflow unchanged:

```sh
# 1. Cancel the stuck run (idempotent if already cancelled)
gh run cancel 24740882149 --repo SchwartzKamel/azure-openai-cli

# 2. Rerun failed/cancelled jobs — preserves the tag, re-queues osx-x64
gh run rerun 24740882149 --failed --repo SchwartzKamel/azure-openai-cli

# 3. Watch (optional)
gh run watch 24740882149 --repo SchwartzKamel/azure-openai-cli
```

This was executed at `2026-04-22T07:09Z`. The `osx-x64` leg was re-queued
against a fresh runner slot; the tag at `fd4ddc7` is untouched and the
already-published GHCR image is reused by digest.
