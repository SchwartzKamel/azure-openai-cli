### Body

> Hi HN — maintainer of az-ai here.
>
> v2.0.0 is a substrate swap, not a feature release. We pulled ~2,200 lines of hand-rolled chat/tool/retry orchestration and put the CLI down on top of Microsoft Agent Framework (the first-party C# agent library, MIT-licensed). The interesting bits, if you care about shipping agent CLIs:
>
> - **Byte-for-byte v1 compatibility.** Every flag, env var, exit code, and `--raw` output byte is preserved. The hot path that Espanso / AHK users depend on didn't move.
> - **AOT single-file stayed honest.** 8.86 MB → 12.91 MB (1.456×). We had a pre-declared 1.5× regression gate before starting the migration; it cleared without a waiver. Startup p95 is inside a 25% budget on every scenario; parse-heavy actually got faster (0.93× v1).
> - **Opt-in OpenTelemetry.** `--telemetry` or `AZ_TELEMETRY=1` emits spans + per-call cost events to stderr. Off by default, silent under `--raw`, zero overhead when disabled. No vendor phones home.
> - **`--estimate`.** Offline, zero-network cost prediction. Short-circuits before credential resolution, so it's safe to gate CI pipelines on.
> - **Dual-tree install.** v1 (`az-ai`) and v2 (`az-ai-v2`) install side by side through the dual-tree window. No forced migration.
>
> What I'd love feedback on: whether the MAF tool-calling surface feels right as an extension point for a plugin registry (FR-012 in the repo), and whether anyone has opinions on the MCP client design we have in draft (FR-013).
>
