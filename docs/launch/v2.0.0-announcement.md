# Azure OpenAI CLI v2.0.0 — The Framework Underneath

> Release window opens **2026-04-20**. Packaged during the dual-tree window as `az-ai-v2`; v1 stays as `az-ai` until cutover. Full details in the [release notes](../release-notes-v2.0.0.md).

## The quiet upgrade

Picture it: a Tuesday, 8:47 AM. You hit the Espanso trigger that wraps `az-ai --raw` around a clipboard of gnarled markdown, and clean prose lands in the text field. Same as last Tuesday. Same as the Tuesday before. That is the point.

v2.0.0 is not the release where everything looks different. It is the release where **everything looks the same, and the floor underneath is new**. We pulled out roughly 2,200 lines of hand-rolled chat, tool, and Ralph orchestration and set the CLI down on top of [Microsoft Agent Framework](https://github.com/microsoft/agent-framework). Every v1 flag still works. Every env var reads the same. Every exit code is unchanged. `--raw` output is byte-identical. Your `.squad/` memory comes along untouched.

## What's new at a glance

- **[Microsoft Agent Framework under the hood](../release-notes-v2.0.0.md#headline)** — `ChatClientAgent`, `AgentThread`, and MAF's function-tool interface replace the v1 bespoke loop. Streaming, tool calls, and Ralph retries all ride on top.
- **[Persona routing, fully wired](../release-notes-v2.0.0.md#persona-mode)** — `--persona coder` and `--persona auto` now overlay the persona's system prompt, tool allow-list, and `.squad/history/<name>.md` memory. In early v2 previews the flag was parsed and silently ignored. Not anymore.
- **[`--estimate`](../release-notes-v2.0.0.md#--estimate)** — offline, zero-network cost prediction. Short-circuits before credential resolution. Safe to gate CI pipelines on.
- **[Opt-in OpenTelemetry](../release-notes-v2.0.0.md#--telemetry)** — `--telemetry` (or `AZ_TELEMETRY=1`) emits spans and per-call cost events to stderr. Off by default. Silent under `--raw`. Zero overhead when disabled.
- **[Connection prewarming](../release-notes-v2.0.0.md#--prewarm)** — `--prewarm` opens the Azure OpenAI channel in parallel with prompt assembly and buys back most of the MAF cold-start cost on the first interactive call.

## Why it matters

The v1 chat loop was a thing we were proud of — for about eighteen months. Every streaming primitive, every tool-call edge case, every retry heuristic: ours to carry. MAF is a first-party library with a working group behind it. Handing off the orchestration substrate means the next eighteen months of agent work compound instead of accumulate.

The trade is honest. The AOT single-file binary grew from **8.86 MB (v1.9.1) to 12.91 MB (v2.0.0)** — a **1.456× increase**, about +4.05 MB, for MAF plus OpenTelemetry plus the Azure SDK bump. That ratio cleared Bania's 1.5× gate without a waiver. Startup p95 sits inside the ≤25% regression budget: `--version --short` at 1.12× v1, `--help` at 1.23×, parse-heavy actually *faster* at 0.93×. RSS is at or below v1 on every scenario. The hot path Espanso and AHK users depend on is safe.

What stays exactly as it was: the monochrome-by-construction security posture. Shell blocklist, `ArgumentList`-based spawning, HTTPS-only fetch, DNS-rebinding guards, symlink traversal blocks, exact-alias tool matching, SIGINT exit 130 — preserved byte-for-byte. Newman signed off at zero red findings. Licensing is clean: 34 MIT, 4 Apache-2.0, 1 BSD-3-Clause across 39 packages, zero GPL family. The **tarball and container image** carry `NOTICE` and `THIRD_PARTY_NOTICES.md` in-band at 2.0.0; Homebrew / Scoop / Nix manifest updates land in the 2.0.1 packaging sweep.

Persona memory remains the differentiator you will not find in the upstream OpenAI CLIs. `.squad.json` plus per-persona history files on disk. `--persona auto` picks by deterministic keyword match — no model call to decide. See the [persona guide](../persona-guide.md).

## Upgrade in 30 seconds

Both binaries install side by side during the dual-tree window. No migration step required.

```bash
# v1 (stable, pinned):
az-ai    --version --short     # → 1.9.1

# v2 (parallel install):
az-ai-v2 --version --short     # → 2.0.0
```

Both read the same env vars and the same config files. `.squad/` memory transfers untouched. Flip your Espanso trigger from `az-ai` to `az-ai-v2` when you're ready; flip it back if you need to.

Full story: [`docs/migration-v1-to-v2.md`](../migration-v1-to-v2.md). Post-cutover v1 pin instructions (tarball / container / pinned-manifest URL) live in the release notes under [Upgrading / rolling back](../release-notes-v2.0.0.md#upgrading--rolling-back).

## Known limitations

Short list; full version in [Known limitations at 2.0.0](../release-notes-v2.0.0.md#known-limitations-at-200). Worth flagging:

- `--schema <json>` wire enforcement is deferred to 2.1.x (flag is captured, not yet sent as `response_format`).
- Homebrew / Scoop / Nix manifests lag by one release — install via the GitHub Release tarball or the GHCR container until 2.0.1.
- arm64 release matrix rolls forward to 2.0.1. Pin `v1.9.1` if arm64 is your primary platform.
- Malformed `.squad.json` persona names surface as an uncaught `ArgumentException` (exit 134) rather than a clean `[ERROR]` line. Security is intact — path traversal is blocked at the validator — the UX wrap lands in 2.0.1 (FR-021).
- Multi-provider (FR-014) is a spike, not a feature. v2.0.0 remains Azure-OpenAI-only.
