# The Floor Is New

*Azure OpenAI CLI v2.0.0, and the eighteen months that got us here.*

— J. Peterman, dispatch from the launch deck, April 2026

---

Picture it: Seattle, a wet Thursday in November 2024. The `az-ai`
binary was eleven months old, the chat loop weighed in at something
like 2,200 lines of hand-carved C#, and every streaming edge case —
every partial-JSON tool call, every mid-token cancellation, every
retry-with-appended-error — was a thing we owned. Maintained. Fed.

We were, if I am being honest, a little proud of it.

Pride is a precursor, in my experience, to carrying a trunk up four
flights of stairs when the freight elevator has been there all along.

## The trunk

The trunk in this case was a handrolled orchestration layer: tool
registry, streaming multiplexer, Ralph retry loop with accumulated
error context, persona dispatch, `.squad/` memory plumbing. Every
bespoke primitive came with its own test suite, its own edge cases,
its own subtle incompatibilities with each new Azure OpenAI API
version that shipped through 2025. Every model upgrade — `gpt-4o` to
`gpt-4.1` to the `gpt-5.x` family — meant another round of
investigations in the chat loop: *is the streaming framing still what
we think it is? Does this model honor our tool-call cancellation
protocol? Why did the JSON mode regress on Tuesday?*

By late 2025 we had shipped v1.9.1. Stable. Solid. AOT-clean at 8.86
megabytes. And quietly, underneath, the maintenance surface was
accumulating faster than we could amortize it. A one-person open
source project with twenty-five named agents on retainer and a
roadmap that wanted to land persona routing, a cost estimator,
OpenTelemetry traces, and multi-provider support — all at the same
time. The trunk was getting heavy.

Then Microsoft shipped the [Agent
Framework](https://github.com/microsoft/agent-framework) — a
first-party library with a working group behind it — and the decision
made itself.

## The freight elevator

ADR-004 was the moment we said it out loud: we are going to delete the
chat loop. Not refactor it. Delete it.

`ChatClientAgent` replaces the bespoke chat client. `AgentThread`
replaces our session state. MAF's function-tool interface replaces
the tool registry shim. `RunStreamingAsync` replaces the streaming
multiplexer. Ralph becomes a thin loop on top of `AgentThread`
that re-injects accumulated errors and iterates. The squad becomes
persona overlays on top of `ChatClientAgent`.

Roughly 2,200 lines, gone. Replaced by calls into a library with
Microsoft's name on it and a roadmap that compounds with the rest of
the agent ecosystem.

The interesting engineering problem was not the migration itself. The
interesting problem was the promise we made on the way in: **no flag,
env var, config file, or exit code changes behavior for existing
users.** Espanso triggers keep working. AutoHotkey macros keep
working. The CI budget-gate scripts keep working. `--raw` output is
byte-identical. The person who hits `Ctrl+Shift+A` at 8:47 AM and
expects clean prose in the text field ten milliseconds later gets
clean prose in the text field ten milliseconds later.

Every. Single. Contract. Preserved.

## The receipt

You pay for things like this. Honest accounting demands we show the
receipt.

- **Binary grew from 8.86 MB to 12.91 MB** — a 1.456× ratio. Inside
  Kenny Bania's 1.5× gate with 44 kilobytes to spare. The cost is MAF
  host assemblies, OpenTelemetry API and exporters, and a bump to
  `Azure.AI.OpenAI 2.9.1` for modern Responses-API deployments.
- **Startup p95 regressed by at most 23%** on `--help`, 12% on
  `--version --short`. Inside the ≤25% budget. `parse-heavy` is
  actually *faster* than v1 (0.93× mean) — the ParseArgs rework
  vindicated, as The Maestro predicted.
- **RSS is at or below v1** on every measured scenario.
- **488 / 488 v2 unit tests passing.** 29 integration assertions
  across 14 cases, both binaries exercised in parallel through the
  dual-tree window.
- **Zero copyleft in the transitive dependency closure.** Jackie
  Chiles audited 39 packages: 34 MIT, 4 Apache-2.0, 1 BSD-3-Clause.
  The v1 `NOTICE` claimed "all dependencies are MIT" — v2's `NOTICE`
  is corrected to reflect the Apache-2.0 and BSD-3-Clause additions.
  Integrity matters more than a clean one-liner.

The security posture — shell blocklist, `ArgumentList`-based process
spawning, HTTPS-only fetch, DNS-rebinding guard, symlink-traversal
block, exact-alias tool matching, `Ctrl+C` → exit 130 — carries
forward byte-for-byte. Newman signed off at zero red findings.

## What shipped alongside

Three features earned their passage on the v2 boat:

**Persona routing, fully wired.** `--persona coder`, `--persona
auto`, `--persona ralph`. System prompt overlay, tool allow-list,
per-persona `.squad/history/<name>.md` memory, persisted on session
exit. In earlier v2 previews the flag was parsed and silently ignored.
That silence is over.

**`--estimate`.** Offline, zero-network cost prediction. It
short-circuits before credential or endpoint resolution, which means
you can run it in a budget gate with no Azure connectivity and
receive a dollar-and-cents answer in milliseconds. Morty Seinfeld
approved.

**Opt-in OpenTelemetry.** `--telemetry`, `AZ_TELEMETRY=1`,
`--otel`, `--metrics`. Spans and per-call cost events to stderr.
**Off by default. Silent under `--raw`.** Zero allocation on the hot
path when disabled — Frank Costanza and Rabbi Kirschbaum both
insisted, and both were correct.

## The prompt blinks the same way

Here is the part I keep coming back to, in the quiet hour after the
tag is pushed and before the release workflow finishes.

The person who opens a terminal tomorrow morning and types `az-ai-v2
"rewrite this paragraph in plain English"` gets the same prose in the
same amount of time with the same exit code as the person who typed
`az-ai` yesterday. The Espanso trigger fires. The AHK macro fires.
The CI budget gate fires. The cursor blinks in the same place.

Everything they can see is the same.

Everything they cannot see is new.

That is, I think, the release I most want to be responsible for. Not
the one where we shout about what we built. The one where we quietly
replaced the foundation of a small tool that a handful of people rely
on every day, and the only way they notice is that the roadmap for
the next eighteen months suddenly looks possible.

The trunk is on the elevator. The elevator goes up now.
